How to Setup

Crux를 처음 사용하거나 온보딩을 건너뛴 경우, 이 가이드를 따라 설정을 완료할 수 있습니다. 3단계만 완료하면 Codex 세션 분석을 시작할 수 있습니다.

Step 1 — Install the CLI

1

CLI 설치

분석할 머신에 Crux CLI를 설치합니다. CLI는 백그라운드에서 Codex 세션 데이터를 수집합니다.

터미널을 열고 아래 명령어를 실행하세요:

curl -fsSL https://raw.githubusercontent.com/Royaltyprogram/aiops/main/scripts/install.sh | sh

Tip: 설치가 완료되면 crux --version을 실행하여 정상 설치를 확인할 수 있습니다.

Step 2 — Create a CLI Token

2

CLI 토큰 발급

대시보드에서 CLI 인증용 토큰을 생성합니다. 이 토큰은 CLI가 서버와 통신할 때 사용됩니다.

토큰 발급 방법

  1. Crux 대시보드에 로그인합니다.
  2. 상단 탭에서 Settings를 클릭합니다.
  3. Create CLI token 버튼을 클릭합니다.
  4. 생성된 토큰을 복사해둡니다.

Important: 토큰은 생성 시 한 번만 표시됩니다. 반드시 복사해두세요. 토큰은 30일 후 만료됩니다.

Step 3 — Run Setup

3

프로젝트 연결

분석할 레포지토리에서 setup 명령을 실행합니다. 디바이스 등록, 프로젝트 연결, 로컬 세션 업로드가 자동으로 처리됩니다.

분석할 레포지토리 디렉토리에서 실행하세요:

crux setup

실행하면 앞서 복사한 토큰 입력을 요청합니다. 토큰을 붙여넣으면 설정이 완료됩니다.

Tip: 자체 호스팅 서버를 사용하는 경우 crux setup --server https://your-server.com으로 서버 주소를 지정할 수 있습니다.

CLI Commands

설정 완료 후 사용할 수 있는 주요 명령어입니다.

Command Description
crux setup 디바이스 등록 및 프로젝트 연결을 진행합니다.
crux collect 로컬 Codex 세션을 수집하여 서버에 업로드합니다.
crux collect --watch 백그라운드에서 지속적으로 세션을 수집합니다.
crux collect --recent 30 최근 30일간의 세션을 일괄 수집합니다.
crux --version 설치된 CLI 버전을 확인합니다.

Dashboard Guide

대시보드는 4개의 주요 탭으로 구성됩니다.

Reports

수집된 Codex 세션의 분석 리포트를 확인합니다. 모델이 의도를 올바르게 이해했는지, 어디서 잘못 해석했는지를 설명합니다.

Usage Analytics

일별 토큰 사용량, 모델/프로바이더 커버리지, 응답 지연 시간, 도구 사용 패턴 등의 사용 통계를 확인합니다.

Sessions

CLI에서 수집된 원시 Codex 세션 목록입니다. 분석 엔진의 입력 데이터로 활용됩니다.

Settings

CLI 토큰 발급 및 관리, 설정 스냅샷, 최근 활동 기록을 확인합니다.

FAQ

온보딩을 건너뛰었는데 다시 볼 수 있나요?

이 문서의 Step 1~3을 따라하면 온보딩과 동일한 설정을 완료할 수 있습니다. 대시보드에서 직접 토큰을 발급받고 CLI로 setup을 실행하면 됩니다.

토큰을 잃어버렸어요.

대시보드의 Settings 탭에서 새 토큰을 발급받으세요. 이전 토큰은 Settings > Issued CLI tokens에서 취소할 수 있습니다.

여러 프로젝트를 연결할 수 있나요?

네. 각 레포지토리 디렉토리에서 crux setup을 실행하면 됩니다. 동일한 토큰으로 여러 프로젝트를 연결할 수 있습니다.

데이터는 어떻게 수집되나요?

CLI가 로컬 Codex 세션 파일을 읽어 서버에 업로드합니다. crux collect --watch를 사용하면 백그라운드에서 자동으로 수집됩니다. 업로드 없이 수동으로만 수집하려면 crux collect를 직접 실행하세요.
